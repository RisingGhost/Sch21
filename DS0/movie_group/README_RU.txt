Краткое описание
=======

Этот набор данных (ml-latest-small) описывает активность по 5‑звёздочным оценкам и свободному тегированию на сервисе рекомендаций фильмов [MovieLens](http://movielens.org). Он содержит 100836 оценок и 3683 применения тегов для 9742 фильмов. Эти данные были созданы 610 пользователями в период с 29 марта 1996 года по 24 сентября 2018 года. Датасет сгенерирован 26 сентября 2018 года.

Пользователи выбирались случайным образом для включения в набор. Все выбранные пользователи оценили как минимум 20 фильмов. Демографическая информация не включена. Каждый пользователь представлен идентификатором, никакой другой информации не предоставляется.

Данные содержатся в файлах `links.csv`, `movies.csv`, `ratings.csv` и `tags.csv`. Подробности о содержимом и использовании всех этих файлов приведены ниже.

Это *разработческий* (development) набор данных. Поэтому со временем он может меняться и не подходит для публикации совместимых исследовательских результатов. Если это ваша цель, используйте доступные *эталонные* (benchmark) наборы данных.

Этот и другие наборы данных GroupLens доступны для загрузки по адресу <http://grouplens.org/datasets/>.

Содержимое и использование файлов
========================

Форматирование и кодировка
-----------------------

Файлы набора данных записаны в формате [значений, разделённых запятыми](http://en.wikipedia.org/wiki/Comma-separated_values) (CSV) с одной строкой заголовка. Столбцы, содержащие запятые (`,`) экранируются двойными кавычками (`"`). Файлы закодированы в UTF‑8. Если символы с диакритикой в названиях фильмов или значениях тегов (например, Misérables, Les (1995)) отображаются некорректно, убедитесь, что любая программа, читающая данные (текстовый редактор, терминал или скрипт), настроена на UTF‑8.


Идентификаторы пользователей
--------

Пользователи MovieLens выбирались случайным образом для включения в набор. Их идентификаторы анонимизированы. Идентификаторы пользователей согласованы между `ratings.csv` и `tags.csv` (то есть один и тот же id относится к одному и тому же пользователю в обоих файлах).


Идентификаторы фильмов
---------

В набор включены только фильмы с хотя бы одной оценкой или тегом. Эти идентификаторы фильмов согласованы с теми, что используются на сайте MovieLens (например, id `1` соответствует URL <https://movielens.org/movies/1>). Идентификаторы фильмов согласованы между `ratings.csv`, `tags.csv`, `movies.csv` и `links.csv` (то есть один и тот же id относится к одному и тому же фильму во всех четырёх файлах данных).


Структура файла с оценками (ratings.csv)
-----------------------------------------

Все оценки содержатся в файле `ratings.csv`. Каждая строка этого файла после строки заголовка представляет одну оценку одного фильма одним пользователем и имеет следующий формат:

    userId,movieId,rating,timestamp

Строки внутри файла отсортированы сначала по userId, затем, внутри одного пользователя, по movieId.

Оценки выставляются по 5‑звёздочной шкале с шагом в половину звезды (0.5 — 5.0).

Метки времени (timestamp) — это количество секунд с полуночи по всемирному координированному времени (UTC) 1 января 1970 года.


Структура файла с тегами (tags.csv)
-----------------------------------

Все теги содержатся в файле `tags.csv`. Каждая строка этого файла после строки заголовка представляет один тег, применённый к одному фильму одним пользователем, и имеет следующий формат:

    userId,movieId,tag,timestamp

Строки внутри файла отсортированы сначала по userId, затем, внутри одного пользователя, по movieId.

Теги — это созданные пользователями метаданные о фильмах. Обычно каждый тег — это одно слово или короткая фраза. Смысл, ценность и назначение конкретного тега определяются каждым пользователем самостоятельно.

Метки времени (timestamp) — это количество секунд с полуночи по UTC 1 января 1970 года.


Структура файла с фильмами (movies.csv)
---------------------------------------

Информация о фильмах содержится в файле `movies.csv`. Каждая строка этого файла после строки заголовка представляет один фильм и имеет следующий формат:

    movieId,title,genres

Названия фильмов вводятся вручную или импортируются из <https://www.themoviedb.org/> и включают год выхода в скобках. В названиях возможны ошибки и несоответствия.

Жанры перечислены через вертикальную черту и выбираются из следующего списка:

* Боевик
* Приключения
* Анимация
* Детский
* Комедия
* Криминал
* Документальный
* Драма
* Фэнтези
* Фильм-нуар
* Ужасы
* Мюзикл
* Детектив
* Мелодрама
* Научная фантастика
* Триллер
* Военный
* Вестерн
* (жанры не указаны)


Структура файла ссылок (links.csv)
---------------------------------------

Идентификаторы, которые можно использовать для связи с другими источниками данных о фильмах, содержатся в файле `links.csv`. Каждая строка этого файла после строки заголовка представляет один фильм и имеет следующий формат:

    movieId,imdbId,tmdbId

movieId — идентификатор фильмов, используемый на <https://movielens.org>. Например, у фильма «Toy Story» ссылка <https://movielens.org/movies/1>.

imdbId — идентификатор фильмов, используемый на <http://www.imdb.com>. Например, у фильма «Toy Story» ссылка <http://www.imdb.com/title/tt0114709/>.

tmdbId — идентификатор фильмов, используемый на <https://www.themoviedb.org>. Например, у фильма «Toy Story» ссылка <https://www.themoviedb.org/movie/862>.

Использование перечисленных выше ресурсов регулируется условиями каждого провайдера.


Кросс-валидация
----------------

В предыдущих версиях набора данных MovieLens включались либо заранее вычисленные кросс‑фолды, либо скрипты для выполнения этих вычислений. Сейчас мы больше не поставляем ни то, ни другое вместе с набором данных, так как большинство современных инструментов содержит эту функцию из коробки. Если вы хотите узнать о стандартных подходах к вычислению кросс‑фолдов в контексте оценки рекомендательных систем, см. [LensKit](http://lenskit.org) — там вы найдёте инструменты, документацию и примеры открытого кода.
