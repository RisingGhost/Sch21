## Глава IV

### Упражнение 00. Виртуальная среда

- Каталог для сдачи: `ex00/`.
- Файлы для сдачи: `venv.py` и папка с вашей виртуальной средой.
- Разрешенные функции: `import os`.

**Библиотеки** (или **пакеты**) — одна из причин, почему программирование стало доступнее. Никогда еще не было так легко начать и быстро увидеть результат. Некоторые программисты пишут код, который могут переиспользовать другие. Многие Python‑библиотеки — open source, то есть доступны всем. Нет нужды заново писать существующие классы, методы или функции: вы можете их переиспользовать. Нужно лишь запустить команду `sudo pip install`. Или погодите…

Такой способ установки Python‑пакетов считается плохой практикой. Устанавливая их описанным выше способом, вы ставите их в системную версию Python. Python на вашей машине нужен не только для программирования, но и для запуска системных утилит. Установка внешних пакетов таким образом может испортить систему. Почти никогда не нужно использовать `sudo pip install`.

Лучше использовать **виртуальные среды**. Думайте о них как о собственной песочнице, где можно делать что угодно. Если что‑то сломается, это затронет только песочницу. На вашей машине должен быть предустановлен пакет `virtualenv`. Если нет — обратитесь к администраторам или установите сами, если работаете на личном компьютере. Мы будем использовать его в следующих упражнениях и проектах.

Это упражнение простое — для разогрева и знакомства с концепцией виртуальных сред. Что нужно сделать:

1. Создайте виртуальную среду с именем вашего никнейма, используя Python 3. Вы будете работать в этой среде на протяжении курса.
2. Активируйте её.
3. Запустите Python 3 из терминала.
4. Выведите имя виртуальной среды с помощью библиотеки `os`.
5. Напишите небольшой Python‑скрипт, который делает то же самое при запуске из командной строки:
    ```
    $ ./venv.py
    Your current virtual env is /Users/McShtuder/shtuder
    ```
6. Деактивируйте среду.
7. Запустите скрипт снова.

Если при деактивации вы получаете KeyError, значение None или исключение — подумайте, почему это произошло. Исправлять это в данном упражнении не нужно, но будьте готовы объяснить, что случилось.

## Глава VII

### Упражнение 03. Очень красивый суп

- Каталог для сдачи: `ex03/`.
- Файлы для сдачи: `financial.py`.
- Разрешенные функции: без ограничений.

Итак, в предыдущем упражнении вы установили две библиотеки. Теперь поработаем с одной из них — **BeautifulSoup**. Она полезна, когда нужно распарсить сайт без API, как это было с HeadHunter в День 00. Проблема в том, что при парсинге страницы вы получаете не только полезную информацию, но и HTML‑разметку, с которой неудобно работать. Этот пакет помогает ориентироваться в различных HTML‑блоках и классах, упрощая извлечение нужных данных. Имейте в виду, что сам по себе это не парсер; он лишь помогает ориентироваться в HTML‑ или XML‑«хаосе» (то есть в среде нужно установить выбранную вами HTTP‑библиотеку).

В этом упражнении вы будете парсить Yahoo Finance. Да, у него есть API, но для учебных целей забудем об этом. Откройте [страницу вроде этой](https://finance.yahoo.com/quote/msft/financials?p=msft) и получите данные для конкретного поля конкретной компании.

Напишите Python‑скрипт, который:
- принимает тикер и поле таблицы в качестве аргументов (например, MSFT, Total Revenue);
- возвращает кортеж с запрошенной информацией;
- особое условие: добавьте в скрипт «сон на 5 секунд» (пригодится позже).

Пример:
```
$ ./financial.py 'MSFT' 'Total Revenue'
('Total Revenue', '134,249,000', '125,843,000', '110,360,000',
'89,950,000', '85,320,000')
```

Если URL не существует — выбросьте исключение. То же самое — если запрошенное поле отсутствует.

## Глава VIII

### Упражнение 04. Профилирование

- Каталог для сдачи: `ex04/`.
- Файлы для сдачи: `financial.py`, `financial_enhanced.py`, `profiling-sleep.txt`, `profiling-tottime.txt`, `profiling-http.txt`, `profiling-ncalls.txt`.
- Разрешенные функции: без ограничений.

Маловероятно, что вы будете писать код, идеально лишённый возможностей для улучшений. Часто нужно понять, почему скрипты работают не так быстро, как хотелось бы. Для этого есть профилировщики. По Википедии, профилирование — это вид динамического анализа программ, измеряющий пространственную или временную сложность программы, использование отдельных инструкций, частоту и длительность вызовов функций и т. п. Чаще всего профилирование используют для оптимизации программ.

Помните скрипт из предыдущего упражнения? Давайте его оптимизируем. Даже если вы гуру программирования, одна из конструкций была не слишком эффективной (мы намеренно попросили сделать так).

1. Примените `cProfile` к вашему `financial.py`, чтобы получить таблицу используемых функций, отсортированную по убыванию общего времени выполнения. Сохраните её в файл `profiling-sleep.txt`.

    ![1](misc/images/1.png)

2. Удалите строку с `time.sleep(5)` из скрипта и запустите профилирование снова. Новая таблица не должна содержать встроенный метод `time.sleep()`. Сохраните её в `profiling-tottime.txt`.
3. Попробуйте другую HTTP‑клиентскую библиотеку и посмотрите, станет ли скрипт быстрее. Новый скрипт сохраните как `financial_enhanced.py`. Результаты профилирования — в `profiling-http.txt`.
4. Получите ту же таблицу, но отсортированную по числу вызовов по убыванию. Полезно знать, какие функции стоит оптимизировать, чтобы они вызывались реже. Сохраните таблицу в `profiling-ncalls.txt`.
5. На этот раз используйте библиотеку `pstats`. Отсортируйте по совокупному времени и получите топ‑5. Сохраните в `pstats-cumulative.txt`.

## Глава IX

### Упражнение 05. PyTest

- Каталог для сдачи: `ex05/`.
- Файлы для сдачи: `financial_test.py`.
- Разрешенные функции: без ограничений.

Скорость работы скрипта — не единственная забота. Он может изначально работать не так, как задумано. Чтобы убедиться, что скрипт работает корректно, нужны модульные тесты. Например, подавайте разные входные данные и проверяйте, что результаты соответствуют ожиданиям.

1. В `ex03`, мы уверены, вы использовали одну или несколько функций. Для каждой функции создайте не менее трёх тестов с использованием библиотеки PyTest. Проверьте, что скрипт возвращает корректную информацию для каждого запроса:
    - Если запросить Total Revenue, возвращается ли совокупная выручка для заданного тикера?
    - Является ли возвращаемый тип кортежем?
    - Если ввести некорректное имя тикера, возникает ли исключение?
2. Вынесите тесты в код вашего `financial.py`. Сохраните файл в вашей директории как `financial_test.py`. Запустите PyTest. Тесты должны пройти. Если нет — доработайте скрипт.
